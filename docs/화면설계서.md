# 화면 설계서

## 화면 도출 방법

기능 목록을 보고 **"이 기능을 사용하려면 어떤 화면이 필요한가?"** 를 생각하면 됩니다.

| 기능 | 필요한 화면 |
| --- | --- |
| 로그인 (아이디·비밀번호, 정지 계정 차단) | 로그인 페이지 |
| 직원카드·출퇴근, 공지 3건, 미니캘린더, 생일, 날씨, 결재 현황 요약 | 대시보드(메인) 페이지 |
| 개인·팀·회사 일정 등록/조회 | 캘린더 페이지 |
| 공지 목록·상세 조회, 작성·수정·삭제(관리자) | 공지사항 페이지 |
| 자료 목록 조회, 업로드, 다운로드, 삭제 | 자료실 페이지 |
| 결재 기안 작성, 받은/보낸/참조 문서함, 승인·반려 | 전자결재 페이지 |
| 부서별 직원 목록, 직원 상세, 채팅 연결 | 조직도 페이지 |
| 채팅방 목록, 1:1·그룹 실시간 대화, 새 대화 시작(부서→직원 선택) | 실시간 채팅 페이지 |
| 본인 인적사항 조회·수정, 비밀번호 변경 | 마이페이지 |
| 본인 월간 출결 현황(출근율·지각·연차) 조회 | 출결 현황 페이지 |
| 계정 등록·수정·정지 복구, 비밀번호 초기화 | 관리자 - 계정·인사정보 관리 페이지 |
| 전 직원 날짜별 출퇴근 기록 직접 수정 | 관리자 - 직원 출결 관리 페이지 |

→ **총 12개 화면** 필요

> 이 그룹웨어는 로그인하지 않으면 로그인 화면 외에는 아무 것도 볼 수 없습니다(`checkAuth()` 실패 시 전 화면이 로그인으로 강제 이동). 그래서 맛집 서비스처럼 "비회원이 둘러볼 수 있는 화면"이 따로 없고, 대신 **일반 직원 화면**과 **관리자 전용 화면(2개)** 으로 나뉩니다.
>

## 1. 공통 레이아웃

모든 화면(로그인 제외)에서 반복되는 공통 영역을 먼저 정의합니다: 상단 헤더 + 좌측 사이드바 + 중앙 콘텐츠 영역 + 모달 오버레이 + 토스트 알림.

![](img/common_layout.png)

**일반 직원 로그인 후 헤더/사이드바**

- 헤더: 로고, 실시간 채팅 아이콘(안읽음 뱃지), 내 정보(아바타·이름·부서/직급), 로그아웃
- 사이드바: 대시보드 / 일정 캘린더 · 전자결재 · 공지사항 · 자료실 / 조직도 · 실시간 채팅 / 출결 현황 / 마이페이지
- 관리자 메뉴 그룹은 숨김 처리됨

![](img/header_employee.png)

**관리자 로그인 후 헤더/사이드바**

- 헤더는 동일함
- 사이드바 하단에 **관리자** 그룹(계정·인사정보 관리 / 직원 출결 관리)이 추가로 노출됨

![](img/header_admin.png)

### 화면 08 — 실시간 채팅 페이지

**파일**: `chat.html` · **예정 URL**: `/chat` (실시간 메시지는 WS `/ws-stomp`, 구독 `/topic/room/{roomId}`, 발행 `/app/chat/{roomId}`)

**접근 권한**: 로그인한 모든 직원 (본인이 속한 채팅방만)

![](img/13_chat.png)

**구성 요소**

| 요소 | 설명 |
| --- | --- |
| 검색창 | 대화 상대·대화방 이름 검색 |
| 필터 탭 | 전체 / 개인(1:1) / 그룹 |
| 채팅방 목록 | 아바타(그룹은 별도 색상, 1:1은 접속 상태 초록 점), 마지막 메시지 미리보기·시각, 안읽음 뱃지 |
| 새 대화 시작 버튼 | 부서 선택 → 직원 선택 모달 오픈, 1명 선택 시 1:1방, 2명 이상 선택 시 그룹방 생성 |
| 메시지 스레드 | 날짜 구분선, 좌/우 말풍선(텍스트·파일), 읽음 표시, 타이핑 인디케이터 |
| 메시지 입력창 | 텍스트 입력 + 파일 첨부, 전송 |

> 새 대화 시작 모달이 열린 상태 참고: `html/13-1_chat_new_modal.html` (조직도·전자결재 참조 대상 선택과 동일한 "부서 클릭 → 오른쪽 직원 목록 필터 → 체크" 패턴)
>
> 조직도 화면(07)의 직원 상세 모달에도 "채팅하기" 버튼이 있어, 그 직원과의 1:1 채팅방으로 바로 진입할 수 있다 (`html/07-1_org_member_modal.html` 참고).

## 3. 화면 이동 흐름도

같은 사이트라도 **누가 로그인했느냐**에 따라 보이는 메뉴와 갈 수 있는 화면이 달라집니다.

그래서 화면 이동 흐름도를 **① 비로그인 → ② 로그인한 일반 직원 → ③ 관리자** 세 가지로 나누어 그립니다.

> **용어 정리**
>
> - **비로그인**: 로그인하지 않은 상태. 이 그룹웨어는 사내 전용 시스템이라 비로그인 상태로 볼 수 있는 화면이 로그인 화면뿐입니다
> - **로그인한 일반 직원**: 로그인한 사원·팀장·부서장. 본인 업무(근태·일정·자료·결재·채팅·마이페이지) 처리 가능. 팀장·부서장은 결재선에 오르면 승인 권한이 추가로 생김
> - **관리자(Admin)**: 시스템 운영자. 계정·인사정보와 전 직원 출결 기록을 관리하는 전용 화면 2개를 추가로 사용

---

### 3-1. 비로그인 상태

```
[아무 URL 접근] ──(checkAuth 실패)──→ [로그인 페이지]
                                            │
                                            └─ 로그인 성공 → [대시보드]  ← 이후 로그인 회원 흐름으로 전환
```

**핵심**: 비로그인 상태에서 존재하는 화면은 로그인 화면 하나뿐입니다. 그 외 URL은 전부 로그인 화면으로 강제 이동됩니다.

---

### 3-2. 로그인한 일반 직원

```
[대시보드]  ← 헤더: [채팅] 홍길동님 [마이페이지] [로그아웃]
   │
   ├─ 출근/퇴근 버튼 ───────────→ (근태 상태 즉시 갱신, 같은 화면)
   ├─ 공지 클릭 ────────────────→ 공지 상세 모달(열람만)
   │
   ├─ [사이드바] 일정 캘린더 ────→ [캘린더]
   │                                  └─ 날짜 칸 클릭 → 일정 등록 모달 → 등록 → [캘린더]
   │
   ├─ [사이드바] 전자결재 ───────→ [전자결재]
   │                                  ├─ 서식 선택 → 기안 작성 폼 → 상신 → [보낸 기안함]
   │                                  ├─ 받은 결재함 → 문서 클릭 → 상세 모달
   │                                  │                 └─ 승인/반려 버튼 ← 본인이 현재 단계 담당자일 때만 노출
   │                                  └─ 참조 문서함 → 문서 클릭 → 상세 모달(열람만)
   │
   ├─ [사이드바] 공지사항 ───────→ [공지사항] → 목록 클릭 → 상세 모달(열람만, 작성·수정·삭제 버튼 없음)
   │
   ├─ [사이드바] 자료실 ─────────→ [자료실] → 업로드 모달 / 다운로드 / 본인 업로드 파일만 삭제 가능
   │
   ├─ [사이드바] 조직도 ─────────→ [조직도] → 멤버 클릭 → 상세 모달 → 채팅하기 → [실시간 채팅(해당 직원과의 1:1방)]
   │
   ├─ [사이드바] 실시간 채팅 ─────→ [실시간 채팅] → 채팅방 클릭 → 메시지 스레드 열람·전송(WebSocket)
   │                                  └─ 새 대화 시작 → 부서 선택 → 직원 선택(1명=1:1 / 2명 이상=그룹) → 대화방 생성
   │
   ├─ [사이드바] 출결 현황 ──────→ [출결 현황] (본인 근태 캘린더)
   │
   ├─ [헤더] 마이페이지 ─────────→ [마이페이지] → 정보 수정 / 비밀번호 변경
   │
   └─ [헤더] 로그아웃 ───────────→ [로그인 페이지]
```

**핵심**: 결재 승인/반려 버튼은 **본인이 현재 결재 단계 담당자일 때만**, 자료 삭제 버튼은 **본인이 업로드한 파일에만** 노출됩니다. 공지 작성·수정·삭제는 일반 직원에게는 아예 보이지 않습니다.

---

### 3-3. 관리자인 경우 (Admin)

관리자는 일반 직원이 하는 모든 기능을 그대로 사용할 수 있고, **계정·인사정보 및 전 직원 출결을 관리하는 전용 화면 2개**가 사이드바 하단에 추가됩니다.

```
[대시보드] (관리자 계정)  ← 사이드바 하단에 "관리자" 그룹 추가 노출
   │
   ├─ (일반 직원이 하는 모든 기능 사용 가능)
   │
   ├─ [사이드바] 계정·인사정보 관리 ──→ [관리자 - 계정관리]
   │                                        ├─ 신규 사원 등록 → 등록 모달 → 저장 (사번 자동 부여)
   │                                        ├─ 수정 → 등록 모달(기존 값 채움) → 저장
   │                                        ├─ 계정 정지 / 정지 복구
   │                                        └─ PW 리셋 → 초기 비밀번호(사번 동일)로 강제 초기화
   │
   └─ [사이드바] 직원 출결 관리 ────────→ [관리자 - 출결관리]
                                              └─ 날짜 선택 → 직원별 출근/퇴근 시각·상태 직접 수정 → 저장
```

> **접근 제한**: 관리자가 아닌 계정이 주소창에 직접 `/admin` 을 입력해도 관리자 메뉴 자체가 보이지 않으며, 백엔드 구현 시에는 인터셉터로 서버 단에서도 차단할 예정입니다.

**핵심**: 관리자는 *본인 소관이 아니어도* 전 직원의 계정과 출결 기록을 직접 수정할 수 있습니다. 일반 직원은 본인 것만, 관리자는 전체를 관리합니다.

## 정리

- 기능 목록에서 **"이 기능을 쓰려면 어떤 화면이 필요한가?"** 를 생각하면 화면을 도출할 수 있다
- 이 그룹웨어는 로그인 없이는 볼 수 있는 화면이 없어서, 비회원용 흐름 대신 **일반 직원 / 관리자** 두 축으로 권한이 나뉜다
- 공통 레이아웃(헤더·사이드바)을 먼저 정의하면 각 화면 설계가 편해진다
- 모달로 열리는 하위 상태(상세보기·작성폼 등)는 `html/` 폴더에 `01-1`, `06-2` 같은 별도 파일로 참고 화면을 만들어두었다
- 화면 이동 흐름도를 그려두면 개발 중 놓치는 연결고리(승인 권한, 본인 소유 여부 등)를 미리 파악할 수 있다
